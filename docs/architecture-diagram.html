Communication Patterns

🌐 Public Request Path

Client → Cloudflare (TLS + WAF) → bl-gateway (auth + rate limit + route) → target service → response

Gateway adds: X-Request-ID, X-Account-ID, X-RateLimit-* headers

🔒 Internal Service-to-Service

Service → Service (direct HTTP over 10.0.0.0/16 private network)

No gateway overhead. Internal admin tokens for auth.

📨 Async (Event-Driven)

Service → Redis Streams → Consumer (bl-flow workers, bl-ai-api BullMQ)

Used for: flow execution, KB ingestion, non-blocking operations

🗄️ Database Access

Each service → PostgreSQL (own schema, direct SQL)

Schema ownership: each service WRITES only to its schema, can READ others
